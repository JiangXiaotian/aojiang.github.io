个人网站搭建记录(一) -- 工具选择

发表于 2017-12-21 | 阅读次数: |

以下是我搭建个人网站的简要记录, 电脑为 mac 系统。

静态网站与动态网站的选择

网站可分为静态网站和动态网站, 静态网站非常适合专注于内容, 例如博客; 而动态网站以数据库技术为基础, 和用户的交互能力强, 动态网站有个很有名的 wordpress 的博客平台。 结合自己实际情况, 我选择功能较弱、较简单的静态网站。

静态网站生成工具的选择

网上有很多有名的静态网站生成工具, 这些工具能从简单的纯文本文件生成一个网站。 常用的文本格式有 reStructuredText(rst) 和 Markdown。
有人列举了网络上排名前十的工具, 排名并不一定准确, 仅做参考:

  1. Jekyll - GitHub Pages 的构建工具（Ruby 语言。
  2. Octopress - 基于 Jekyll 的博客生成工具, 它简化了 Jekyll 的操作, 可以让你更舒服的创作。
  3. Hexo - 用 Node.js 编写的博客框架, 支持所有的 GitHub Markdown 特性。
  4. Hugo - 使用 go 语言编写, 并且使用 Markdown 语法。
  5. Pelican - 使用Python编写的静态网站生成工具, 支持用 reStructuredText, Markdown, 和AsciiDoc 创作网站内容。
  6. Middleman - 中间人, 一个使用 Ruby 编写的静态网站生成工具。
  7. Metalsmith - 使用 Node.js 编写。
  8. DocPad - 支持 CoffeeScript、Ruby、PHP、Stylus 等等。
  9. Wintersmith - 使用 Node.js 编写。
  10. Cactus - 使用 Python 和 Django 模版系统制作的静态网站生成工具。
    更多资料参考:top 10 排名。

因为熟悉一点 python 和 sphinx, 通过简单的筛选, 我最先选定的是用 Pelican+reStructuredText。 然后找到一个不错的 Pelican 主题(就是网站的整体风格) – elegant, 效果可参考孤逐王的博客。
但是在找的过程中发现了更喜欢的 hexo 主题 - Next, 效果可参考XueCat的博客。 最终还是选用了 Hexo, 采用 Next 主题, markdown 写文档。 嗯, 怎么舒服怎么来。

另外, hexo 这两个主题我也很喜欢:

  • yilia, 效果可参考Litten的博客。
  • raytaylorism, 效果可参考林寿山的博客。

总结

我选择用静态网站搭建博客, Hexo + Next主题 + Markdown 文本格式。
